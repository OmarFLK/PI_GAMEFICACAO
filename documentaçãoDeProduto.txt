Projeto: QuimLab
Instituição parceira: ETEC Júlio de Mesquita

1. Visão geral do projeto

O QuimLab é um sistema desktop em Java voltado para gamificação do ensino de química e laboratório.
O projeto foi organizado com backend em JDBC/MySQL, uma camada frontend em Swing e o servidor em nuvem 
para o MySQL usando o railway.

2. Objetivo do sistema

Entregar uma experiência visual navegável para login, registro, homes, gameplay, ranking, estatísticas,
perfil e gerenciamento de perguntas.

3. Público-alvo

* alunos do ensino médio técnico em Química
* professores que administram perguntas e acompanham desempenho

4. Estrutura atual do projeto

* src/App.java = ponto de entrada
* src/backend = camada já existente do projeto original
* src/frontend = camada visual em Swing
* lib = dependências do projeto
* build/classes = saída dos arquivos compilados .class

5. Padrões do time encontrados no código

* classes curtas e objetivas
* nomes em português
* comentários simples explicando intenção local
* uso de Swing com estrutura direta
* organização por responsabilidade prática

6. Decisão de trabalhar só o frontend

Nesta etapa o foco é somente o frontend.
Não haverá integração real com backend, banco de dados, DAO, service ou JDBC.

13. Como navegar no sistema

* o projeto abre em LoginTela
* no login, o combo de perfil mockado decide a home de destino
* aluno navega por jogo, ranking, estatísticas e perfil
* professor navega por teste, formulário e gerenciamento de perguntas
* todas as telas de apoio têm botão de voltar funcional
* logout volta para LoginTela

14. Como rodar o frontend

Compilar:

`mkdir -p build/classes`

`javac -cp 'lib/*' -d build/classes $(find src -name '*.java' | tr '\n' ' ' )`

Executar:

`java -cp 'build/classes:lib/*' App`

Observação:

No ambiente de desenvolvimento local com interface gráfica o sistema abre normalmente.
No sandbox desta sessão não foi possível abrir a janela por falta de servidor gráfico X11.

Organização dos arquivos:

* os arquivos `.java` continuam dentro de `src/`
* os arquivos `.class` devem ficar apenas em `build/classes/`
* isso evita misturar fonte com compilado
